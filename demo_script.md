# 데모 영상 스크립트 (3분)

> AX 프런티어 챌린지 결선 제출용 / 위원회 운영 통합 자동화 플랫폼

## 핵심 메시지 (한 줄)

**"4-6일 걸리던 위원회 운영을 1.5시간으로. 행정 담당자는 클릭 한 번, 일반 위원은 로그인 없이."**

## 전체 구성 (180초)

| 구간 | 시간 | 비중 | 목적 |
|------|------|------|------|
| ① 인트로 (문제 제기) | 0:00 – 0:20 | 20s | 청중에게 문제 공감 형성 |

### ③ 0:30 – 1:30  운영자 시점 시연 (60초) ━━━━━━━━━━━━━━━━━━

#### 0:30–0:38 (8s) — 로그인

**화면**: `http://localhost:3000/admin/login`
- `haksa1` / `demo1234` 타이핑 → 로그인
- (마우스가 "처음이신가요? 가입 신청하기" 링크 위에 잠시 hover)

**내레이션**:
> "학사팀 담당자가 로그인합니다. 위원회 운영 권한이 있는 담당자만 로그인. **일반 위원은 로그인이 필요 없습니다.**"

**자막**: `운영자만 로그인 / 가입 → 슈퍼관리자 승인 후 활성`

#### 0:38–0:50 (12s) — 대시보드 보여주기

**화면**: 대시보드. 히어로 카드의 큰 숫자에 마우스 hover
- 화면 위에 강조 동그라미: 누적 절감 시간, 97% 단축
- 우측 KPI 5개 카드도 잠시 강조

**내레이션**:
> "대시보드에서 본인 위원회 현황과 **누적 행정 시간 절감**을 한눈에 확인합니다. **'기존 50시간 → 현재 1.5시간, 97% 단축.'**"

**자막**: `▲ 시간 절감 자동 산출`

#### 0:50–1:10 (20s) — 회의 생성 (조건 입력)

**화면**: 학사운영위원회 클릭 → "새 회의 일정 잡기"
- 회의명: "2026-1학기 정기 학사운영위원회" 타이핑
- 날짜 범위: 5/20 ~ 5/29
- 제외 날짜: 5/22 입력
- 시간: 오전 10시 ✓, 오후 13/14/15 ✓
- 특정 날짜 다른 시간: `2026-05-23 : 10` 한 줄 입력
- 알림 채널: 이메일 ✓ SMS ✓
- ARS: "12시간 후" 선택
- "회의 생성 및 일정 조율 시작" 클릭

**내레이션** (속도감 있게):
> "회의 조건을 입력합니다. **'5월 20일부터 29일 중, 22일 제외, 오전 10시 또는 오후 1-3시 시작. 23일만 10시 가능. SMS와 이메일 둘 다 발송, 응답 없으면 12시간 후 ARS 자동 전화.'** 클릭 한 번."

**자막**: `▼ 날짜 범위·제외일·시간 제한·ARS — 정밀 조건 설정`

#### 1:10–1:30 (20s) — 응답 매트릭스 + 시간표 자동 인식

**화면**: 회의 상세 페이지 진입
- 진행 단계 4단계 타임라인 강조 (1단계만 활성)
- 후보 일정 매트릭스로 스크롤 — 6개 슬롯 × 5명 위원 표 보여주기
- 마우스를 'X시' 배지에 hover하여 툴팁 강조 ("시간표 자동 인식: 컴퓨터공학개론 09:00-10:15")
- 위원별 응답 링크 목록 보여주기

**내레이션**:
> "시스템이 **후보 슬롯 6개를 자동 생성**하고, 위원 5명에게 응답 링크를 **동시 발송**합니다. 한성대 시간표 시스템을 **자동 크롤링**해서 교수님 강의 시간과 겹치는 슬롯은 **'불가능'으로 자동 표시.**"

**자막 (화면 하단 강조)**: `🤖 시간표 자동 인식 — 교수 사번만 등록하면 끝`

#### 1:45–2:00 (15s) — 모바일 응답 화면

**화면**: 모바일 뷰포트로 `/avail/{token}` 페이지

## 사전 준비 체크리스트

### A. 데모 데이터 보강 (녹화 전 필수)
- [ ] 시드를 보강해서 차트가 풍성하게 보이도록: 추가 회의 5-10건, 완료 상태로 일부, PDF 일부 생성
- [ ] 학사운영위원회에 회의 1건은 '응답 수집중' 상태로 (응답 매트릭스 시연용)
- [ ] 한 회의는 '완료' 상태로 (PDF 다운로드 시연용)
- [ ] 알림 로그가 50건 이상 쌓이도록 (도넛 차트 다양화)

### B. 환경 셋업
- [ ] Chrome 창 크기 1280x800 고정 (녹화 일관성)
- [ ] 즐겨찾기 모음 숨기기 (Ctrl+Shift+B)
- [ ] 시크릿 창 사용하지 말 것 (대시보드 너무 깨끗하면 데모 같지 않음)
- [ ] 모바일 시연은 같은 Chrome 의 dev tools → Toggle device toolbar → iPhone 12 Pro
- [ ] 한성대 시간표 mock 데이터가 정상 작동하는지 미리 확인 (services/timetable.js)

### C. 녹화 도구
- **화면 녹화**: OBS Studio (무료, Windows/Mac) 또는 Xbox Game Bar (Win+G)
- **모바일 녹화**: Chrome dev tools 모바일 뷰 → OBS 로 캡쳐 (실제 휴대폰 안 써도 됨)
- **편집**: DaVinci Resolve (무료) / Clipchamp (Windows 11 기본) / CapCut
- **자막 생성**: 클로바 노트 (자동) 또는 직접 작성 후 SRT 파일

### D. 음성
- 마이크: 노트북 내장 마이크는 잡음 심함. **헤드셋 마이크 권장** (게이밍 헤드셋 충분)
- 환경: 카펫 깔린 방 / 옷장 안 / 이불 뒤집어쓰고 녹음하면 음질 ↑
- 녹음 도구: 무료 — Audacity / 윈도우 음성 녹음기
- 페이스: 1분에 약 280-320 글자. 빠르게 들리지만 자막 있어서 OK

### E. 자막 색상/스타일 가이드
- **제목 자막**: 화면 상단 / 흰색 / 굵게 / 24-28pt
- **강조 숫자**: 화면 중앙 또는 우측 / **노란색 또는 에메랄드 그린** / 48pt 이상
- **부연 설명**: 하단 / 회색 / 16-18pt
- 배경에 반투명 검은 박스 (가독성)

## 녹화 순서 추천 (편집 효율 위해)

1. **영상부터 통째로**: 운영자 + 위원 시연을 한 번에 녹화 (실수해도 살릴 수 있음)
2. **음성은 별도**: 영상 보면서 별도 녹음. 1구간씩 끊어서 다시 녹음 OK
3. **자막은 마지막**: 영상 + 음성 sync 맞춘 후 자막 입력
4. **배경 음악**: YouTube Audio Library — 차분한 ambient/corporate 음악 검색 (예: "Inspiring Corporate")
