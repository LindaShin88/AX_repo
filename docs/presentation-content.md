## 🔥 Slide 2 · 문제 정의

**제목**: **회의 1건 4-6일 × 팀당 월 2건 × 60+ 위원회**

**부제**: 보이지 않는 행정 손실의 누적

**본문 (2×2 카드)**:
- 📞 **일정 조율** — 위원 7~15명에게 전화·메일·엑셀로 가능 시간 수집
- 📅 **시간표 확인** — 학사 시스템 일일이 조회
- ✍️ **외부위원 서명** — 메일/방문 요청
- 📄 **회의록 정리** — 수기로 매번 작성

**스케일 공식 (강조 박스)**:
> **회의 1건 4-6일 × 팀당 월 2건 × 60+ 위원회**
> = 연간 **1,000+ 시간** 행정 손실

**발표자 노트**: 현재 한 회의의 준비와 마무리에 평균 4-6일이 소요됩니다. 한 팀에서 한 달에 약 2건의 회의를 진행하고, 한성대에는 60개 이상의 위원회가 있습니다. 이걸 곱하면 연간 1,000시간 이상의 행정 시간이 사라지고 있습니다.

## 💎 Slide 3 · 솔루션 — 4-6일 → 1.5시간

**제목**: **4-6일 → 1.5시간**

**부제**: 모든 역할의 부담을 동시에 줄였습니다

**본문 (4분할)**:

| ⚡ 97% 시간 단축 | 🔗 로그인 불필요 |
|---|---|
| 4-6일 → 1.5시간 | 메일 1회용 링크 |
| 운영자 개입은 시작·확정 2번뿐 | 계정·앱 설치 X |

| 📡 시간표 자동 인식 | ✍️ 외부위원 재방문 0회 |
|---|---|
| 한성대 113개 트랙·전공·학과 1.9초 | 메일에서 손글씨 전자서명 |
| 강의 시간 사전 회피 | PDF 자동 결합 |

**발표자 노트**: 기존 4-6일이 1.5시간으로 압축됩니다. 시간 단축뿐 아니라 운영자, 일반 위원, 외부위원의 부담을 동시에 줄였습니다.

## 🔁 Slide 4 · 4단계 사용자 흐름

**제목**: 회의 운영, 4단계로 끝납니다

**본문 (가로 4카드 + 연결선)**:

| 01 회의 생성 | 02 위원 응답 | 03 일정 확정 | 04 서명·PDF |
|---|---|---|---|
| 운영자 5분 | 위원 1분, 로그인 X | 자동/수동 | 즉시 자동 생성 |
| 후보 일정 자동 + 교원 강의 시간 회피 | 메일/문자 링크 1클릭 | 전원 응답+단일 최고면 자동, 동률이면 알림 | 회의록 + 서명 페이지 자동 병합 PDF |

**스케일 공식 적용**:
> **5분 + 1분 + 즉시 + 자동 = 회의 1건 1.5시간**

**발표자 노트**: 운영자가 손대는 시간은 5분 + 확정 클릭 2번이 전부입니다. 나머지는 시스템이 처리합니다.

## 🚀 Slide 5 · 핵심 기능 ① 한성대 시간표 자동 인식

**제목**: 권한 없이 한성대 **113개 트랙·전공·학과** 1.9초 교차 검색

**부제**: 교원의 강의 시간을 회의 후보에서 사전 제거

**본문 (스토리)**:
- 한성대 OC(Open Class) 시간표 페이지를 익명 크롤링
- 한 교수가 여러 트랙에 강의해도 자동 통합
- 후보 슬롯 생성 시 강의 시간 사전 필터링

**예시 (실제 작동)**:
> 전자트랙 오종택 교수
> → 디지털 신호처리·컴퓨터구조·전자종합설계 **강의 8건 자동 수집**
> → 회의 후보에서 해당 요일·시간 자동 제외

**스케일 공식 적용**:
> **위원당 5-10분 × 위원 15명 = 75-150분**
> → 시스템 **1.9초/전체**

**기술 강조**: `concurrency=6` 병렬 + 정규식 파싱 + 학기별 캐시

**113개 기준**: 한성대 학사 시간표 시스템(`siganpyo_aui.jsp`)의 전공 목록 API가 2026-1학기 기준 반환한 모든 학부 단위 (트랙·전공·학과·일반교양 포함)

**발표자 노트**: 한성대 학사 시간표에 등록된 113개 트랙·전공·학과를 1.9초 안에 병렬 검색합니다. 운영자가 일일이 학사 시스템을 열어볼 필요가 없어집니다.

## 🧩 Slide 6 · 핵심 기능 ② 데이터 자동 분류

**제목**: 복잡한 행정 데이터, 시스템이 분류합니다

**부제**: 동명이인 자동 구분 + 일정 다양성 알고리즘

**본문 (2개 영역)**:

**A. 한성대 디렉토리 자동 채우기**
- 이름만 입력 → 이메일·사번·소속·전화 자동 입력
- **동명이인은 전화번호로 자동 구분**
- 같은 사람의 여러 소속은 자동 병합

**B. 일정 다양성 알고리즘**
- 그리디 방식으로 (날짜 × 시간) 균형 배치
- 시간표 충돌은 점수가 아닌 **사전 필터링**
- 점수 동률 시 새 날짜·시간 우선 채택

**예시 박스**:
> "문혜진" 검색 → 2명 자동 식별
> ① 산업공학트랙 (02-760-4352)
> ② 기업경영트랙 (02-760-4419)
> 운영자는 선택만 하면 됨

**발표자 노트**: 행정에서 가장 흔한 실수가 동명이인 혼동입니다. 본 시스템은 전화번호로 자동 구분해 실수를 원천 차단합니다.

## ✒️ Slide 7 · 핵심 기능 ③ 외부위원 전자서명 + PDF 자동 결합

**제목**: 메일 한 통으로 끝나는 서명 흐름

**부제**: 메일/방문 → 우편 → 회수 → 스캔의 전 과정 제거

**본문 (Before / After)**:

| 단계 | 기존 | 본 시스템 |
|---|---|---|
| 회의록 전달 | 메일/방문 요청 | 메일 링크 즉시 |
| 종이 인쇄 | 필요 | 디지털 그대로 |
| 서명 회수 | 우편/방문 (1-3일) | 손글씨 캔버스 즉시 |
| 스캔 + 저장 | 수작업 | 자동 디지털 저장 |
| 최종 PDF | 수기 정리 30분 | **pdf-lib 자동 병합** |

**기능 디테일**:
- PDF 회의록 + 서명 페이지 → 단일 PDF 자동 생성
- 한글(.hwp) 업로드 시 → 안내 페이지 + 서명 페이지 자동 결합
- 서명일·서명자 정보 자동 기록 (위변조 방지)

**스케일 공식 적용**:
> 외부위원 3명 × 3-5일 = **9-15일** → **즉시 (메일 1클릭)**

**발표자 노트**: 외부위원에게 메일 보내고 회수받는 데 3-5일 걸리던 과정이 메일 링크 한 번으로 끝납니다.

## 🛠 Slide 9 · 기술 스택 + 개발 효율

**제목**: 실제 작동하는 풀스택 시스템

- **메일**: Nodemailer + Gmail/Naver SMTP
- **크롤링**: axios + iconv-lite (EUC-KR 자동 디코딩)
- **배포**: Render + Cloudflare Tunnel

**AX 요소 (개발 프로세스 자체가 AX)**:
- 생성형 AI (Claude Code) 페어 프로그래밍 활용

- 한성대 직원 1인이 풀스택 시스템 완성

**오픈소스 공개**: https://github.com/LindaShin88/AX_repo

## ✨ 발표 팁

### 5-7분 발표 시
- 슬라이드당 30-40초
- 1·2·3·8·10번에 더 시간 할애 (문제·해결·효과·마무리)
- 5·6·7번은 빠르게 핵심만

### 데모 시연
- 발표 중간 또는 마지막에 1-2분 라이브 데모
- 추천 흐름: 회의 생성 → 매트릭스 확인 → 강제 확정 (SMS 텍스트 팝업까지)
