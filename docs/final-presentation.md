## 3️⃣ Q&A 예상 질문 + 답변

### 행정·도입 관점 (총장·보직교수가 물을 가능성)

**Q. 실제 한성대에 도입한다면 비용이 어떻게 되나요?**
> 현재 시스템은 오픈소스로 공개되어 있고, 외부 의존성은 모두 무료 또는 학내 자원입니다. 메일은 학내 SMTP, DB는 SQLite, 호스팅은 클라우드 무료 플랜으로 시작할 수 있습니다. 본격 도입 시에는 한성대 SSO 연동과 학사 시스템 통합만 추가하면 됩니다.

**Q. 외부위원 서명을 메일 링크 한 번으로 받는 게 보안상 안전한가요?**
> 토큰 기반 1회용 링크 방식이고, 토큰은 회의·위원 조합마다 고유합니다. 서명 시점에 위원 식별 정보와 서명 페이지가 PDF에 자동 기록되어 위변조를 추적할 수 있습니다. 실제 운영 시에는 한성대 SSO를 추가해 신원 검증을 한 단계 더 강화하는 것을 권장합니다.

**Q. 기존 학사 시스템과 충돌하거나 부담을 주지 않나요?**
> 외부 크롤링만 사용하고 학사 DB에 쓰기 작업은 일절 하지 않습니다. 시간표 조회는 공개된 Open Class 페이지를 익명으로 보는 방식이라, 일반 학생이 시간표를 조회하는 것과 동일한 부하 수준입니다.

**Q. 다른 부서나 다른 대학으로 확장 가능한가요?**
> 학교마다 시간표 시스템 구조가 다르지만, 크롤러 모듈만 학교별로 교체하면 동일한 자동화 흐름을 적용할 수 있습니다. 위원회 외에도 평의회·심사위원회 등 다자간 일정 조율이 필요한 모든 행정 영역에 확장 가능합니다.

**Q. 운영자가 잘못된 일정을 자동 확정하면 어떡하나요?**
> 자동 확정은 **전원 응답 + 단일 최고 슬롯**일 때만 작동합니다. 응답이 부족하거나 동률이 나오면 자동 확정 없이 운영자에게 알림이 가서, 사람이 최종 판단합니다.

### 기술 관점 (직원·학생이 물을 가능성)

**Q. 1-2주 만에 어떻게 풀스택을 완성했나요?**
> 생성형 AI(Claude Code)와 페어 프로그래밍 방식으로 진행했습니다. AI는 코드 생성·디버깅·한성대 시스템 분석에 사용했고, **사람이 판단해야 하는 부분 — 데이터 흐름, UX, 보안 정책 — 은 직접 검토**했습니다. 이 작업 방식 자체가 AX 프런티어 챌린지의 취지에 부합한다고 생각합니다.

**Q. AI는 정확히 어디에 썼나요?**
> ① 코드 생성과 리팩토링 ② 한성대 학사 시스템의 HTML 구조 분석과 정규식 작성 ③ EUC-KR 인코딩 같은 한국 특수 환경 문제 해결 ④ 디버깅. 단, 최종 코드 결정과 데이터 흐름 설계는 사람이 책임졌습니다.

**Q. 한성대 시간표 크롤링에 권한 문제는 없나요?**
> 공개된 Open Class 페이지(`siganpyo_aui.jsp?viewMode=oc`)는 로그인 없이 누구나 볼 수 있는 학사 시간표 공개 페이지입니다. robots.txt 위반이나 API 남용 없이 일반 브라우저와 동일한 요청만 보냅니다.

**Q. SQLite로 60개 위원회 규모를 감당할 수 있나요?**
> 현재 데이터 규모(위원회·회의·위원 합쳐 수만 행 수준)는 SQLite로 충분합니다. 본격 도입 시에는 PostgreSQL로 마이그레이션할 계획이고, 코드 구조상 DB 어댑터만 교체하면 됩니다.

### 검증·데이터 관점

**Q. 4-6일 → 1.5시간이라는 수치는 어떻게 산출했나요?**
> 한성대 행정 부서 인터뷰 기반 가정값과 본 시스템 실측 수치를 결합했습니다. 외부위원 서명의 경우 우편·방문 왕복 3-5일이 시스템에서는 즉시 처리되므로, 단축 효과는 명확하게 측정됩니다.

**Q. 실제 한성대 위원회에서 써본 적은 있나요?**
> 현재는 데모 단계지만, 시드 데이터에 실제 한성대 위원회 구조(학사운영위원회 등)를 반영했습니다. 본 발표 이후 학내 시범 운영을 진행할 계획입니다.

### 까칠한 질문 (대비용)

**Q. 시중 일정 조율 도구(When2Meet, 캘리)와 차별점이 뭔가요?**
> 일반 일정 도구는 ① 시간표 자동 회피 ② 외부위원 서명 ③ 회의록 PDF 결합이 모두 분리되어 있습니다. 본 시스템은 한성대 환경에 맞춰 **이 세 가지를 한 흐름으로 통합**했고, 특히 학사 시간표 자동 인식과 외부위원 재방문 0회는 일반 도구로는 구현 불가능한 기능입니다.

**Q. 이미 한성대 어디선가 비슷한 시스템을 쓰고 있지 않나요?**
> 현재 한성대 위원회 운영은 부서별로 엑셀·메일·전화를 혼합해서 사용하는 방식이고, 통합 자동화 시스템은 없습니다. 본 시스템은 그 분산된 흐름을 하나로 묶는 첫 시도입니다.
